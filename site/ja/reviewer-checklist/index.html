1つの責任経路を点検する

RPE recordにおいて、責任主体、AI境界、証拠、不確実性、人間への戻り先、修復責任者が見える状態に保たれているかを確認するためのブラウザ向けチェックリスト。

境界: このページは点検補助です。認証、法的レビュー、安全性レビュー、遵法性レビュー、公平性レビュー、本番承認、組織的 endorsement、AI最終責任移転ではありません。

チェックリスト

1. 人間のdecision owner

判断の責任者を特定できるか。

decision_owner を確認し、人間または組織に向いているかを見る。

2. Approval gate

承認権限が明示されているか。

approval_gate を確認する。AI支援が暗黙に承認へ変わっていないことを見る。

3. Stop authority

経路を止められるか。

stop_authority を確認する。介入可能な場所が残っている必要がある。

4. AI境界

AI最終責任がブロックされているか。

assumes_final_responsibility: false と ai_final_responsibility_allowed: false を確認する。

5. 証拠と不確実性

根拠を点検できるか。

evidence logs、source references、human instructions、AI drafts、approval state、uncertaintyを見る。

6. Human return point

争い・不足・不明点はどこへ戻るか。

return_points を確認し、人間または組織のレビューへ戻ることを見る。

7. Repair owner

誰が修復・再オープンできるか。

repair_owner を確認する。誤りや不明点があるとき、経路は再点検可能であるべき。

8. 除外された強い主張

強い主張を含めていないか。

安全、法的、遵法、公平性、本番性、認証などを暗黙に主張していないことを確認する。

コピー用レビューメモ

## RPE reviewer checklist

- [ ] 人間のdecision ownerが明示されている。
- [ ] Approval gateが明示されている。
- [ ] Stop authorityが明示されている。
- [ ] AI最終責任がブロックされている。
- [ ] 証拠とsource referenceが点検可能である。
- [ ] 不確実性と不足文脈が見える。
- [ ] Human return pointが見える。
- [ ] Repairまたはreopening ownerが見える。
- [ ] 強い主張が除外されている: 認証、法的レビュー、安全性レビュー、遵法性レビュー、公平性レビュー、本番承認、組織的endorsement、AI最終責任移転。

レビューメモ:

境界: このチェックリストは点検補助であり、認証や本番承認ではない。

定義と主張の責任は人間の著者に残ります。AIツールはドラフトや実装を支援できますが、責任ある著者にはなりません。
