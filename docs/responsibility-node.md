# Responsibility Node

## Purpose

A Responsibility Node is a point in a Responsibility Pathway where responsibility-related information can be held, transformed, suspended, returned, or repaired.

A node is not necessarily a person.

A node may be a human, AI agent, organization, institution, workflow step, document, interface, database record, model output, audit artifact, or decision point.

## Minimal Definition

A Responsibility Node is any entity or artifact that can participate in a Responsibility Pathway by receiving, preserving, transforming, or returning responsibility-related state.

## Node Capabilities

A node may have one or more of the following capabilities:

- receive
- preserve
- transform
- explain
- escalate
- suspend
- return
- repair
- close

Not all nodes have all capabilities.

## Human Judgment Boundary

Only a human or human-authorized institution can ultimately hold human responsibility in the sense of judgment, commitment, or public accountability.

AI agents and technical artifacts may participate in pathways, but they do not become human-responsible agents by participation alone.

## Node State

A node may carry state related to the eight elements:

- meaning
- authority
- time
- quality
- trust
- reversibility
- value
- cost

## Failure Modes

A node can break or obscure a Responsibility Pathway when it:

- drops context
- hides authority
- erases time order
- lowers quality without visibility
- creates false trust
- removes reversibility
- captures value without cost visibility
- transfers cost without return path

## Boundary

A Responsibility Node is a modeling construct.

It is not a legal person, moral agent, or blame target by default.
