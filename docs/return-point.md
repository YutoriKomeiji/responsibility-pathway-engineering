# Return Point

## Purpose

A Return Point is a designed point where responsibility-related state can return to an entity capable of understanding, judgment, authority, repair, or continuation.

A return point is not merely a checkpoint, approval button, escalation route, or audit log.

It is a point where a pathway becomes returnable.

## Minimal Definition

A Return Point is a Responsibility Node or node relation that allows responsibility-related state to return from an automated, organizational, or technical pathway to a responsible human, organization, institution, or authorized process.

## Returnability

A pathway is returnable when the receiving entity has enough:

- meaning
- authority
- time
- quality of information
- trust calibration
- reversibility
- value visibility
- cost visibility

to understand the state and continue, stop, repair, or accept responsibility for the next action.

## Human Return Point

A Human Return Point is a return point where responsibility-related state returns to human understanding, human authority, and human responsibility capacity.

Human presence alone is not enough.

A human may be present but not capable of responsibility return if:

- the relevant information is hidden
- the decision is already effectively forced
- there is no meaningful time to review
- refusal is punished or unavailable
- the human lacks authority to stop or repair
- the evidence is not reconstructable
- the cost or affected party is invisible

## Return Point Roles

A return point may support:

- explanation
- rejection
- escalation
- suspension
- rollback
- repair
- compensation
- continuation
- closure

## Distinction from HITL

Human-in-the-loop means a human is included in a process.

Human Return Point means responsibility can return to a human-capable state.

These may overlap, but they are not identical.

## Distinction from Audit Log

An audit log records events.

A return point allows an entity to understand, judge, stop, repair, or continue from those events.

Logs may support returnability, but they are not sufficient by themselves.

## Boundary

A return point is a modeling and design concept.

It does not itself prove legal accountability, moral responsibility, safety, or compliance.
