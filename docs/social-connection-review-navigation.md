# Social Connection Review Navigation

This note provides a focused reader path for reviewing social-connection concerns in Responsibility Pathway Engineering.

It is a navigation note only.

It does not introduce schema changes, checker changes, workflow changes, runtime behavior, connector behavior, legal decision logic, compliance certification, safety certification, social acceptance certification, or production authorization.

## Use when

Use this note when a pathway moves beyond an internal model and touches real organizations, affected parties, contracts, policies, legal systems, or cross-border governance.

Use it especially when reviewing:

- approval skip
- responsibility transfer
- evidence logging
- explanation paths
- affected-party visibility
- later review, dispute, repair, or return paths

## Read-first path

Recommended reading order:

1. `docs/validation-checklist.md`
2. `docs/social-connection-exception-cases.md`
3. `docs/evidence-approval-transfer-alignment.md`
4. `docs/ai-judgment-node-task-control.md`
5. `docs/evidence-log.md`
6. `formal/lean/README.md` when structural formalization or evidentiary completeness claims matter

## Review focus

Check whether the pathway preserves:

- evidence needed for later reconstruction
- explanation path when explanation is required
- approval-skip rule when approval is skipped
- responsibility-transfer destination when responsibility moves upward
- affected-party route when affected parties differ from contracting or operating parties
- return, dispute, repair, or reopening route when later challenge is possible
- boundary language that avoids legal, safety, compliance, social acceptance, or production-readiness overclaiming

## Boundary

This note does not decide whether a pathway is legally valid, socially accepted, safe, compliant, fair, or production-ready.

It only connects the current review documents needed to avoid losing social-connection concerns during repository maintenance.
