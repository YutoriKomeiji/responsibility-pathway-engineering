## External reference vs external verification

The article distinguishes external references from external verification. Being introduced on an external site, reposted by an automatic summary site, shown in AI search, mentioned on SNS, or downloaded are external reactions. External verification requires verifier, target, method, reproduction conditions, limitations, falsifiability, and conflict-of-interest disclosure.

Current repository alignment:

- directly supports source-alignment discipline
- supports not treating external mentions or search visibility as validation
- supports not overstating GitHub stars, Zenn references, AI summaries, or article reactions as proof

Repository change decision:

- no validation claim should be based only on external reference
- future documentation should separate reference, adoption, verification, and implementation

## AI summary boundary

Current repository alignment:

- strongly aligned with avoiding overreliance on AI summaries

Repository change decision:
