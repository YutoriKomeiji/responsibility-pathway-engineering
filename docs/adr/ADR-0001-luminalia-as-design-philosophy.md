# ADR-0001: Luminalia as Design Philosophy

## Status

Accepted

## Context

The project needs a stable philosophical layer that explains why Responsibility Pathway Engineering focuses on boundaries, return points, and human judgment rather than automatic decision replacement.

The name Luminalia is used here not as a personality system, but as a light-source metaphor for making boundaries visible.

## Decision

Luminalia is defined in this repository as a design philosophy:

- It illuminates boundaries.
- It does not decide for humans.
- It does not assign blame.
- It does not guarantee correctness, safety, or success.
- It helps responsibility, meaning, and understanding return.

## Consequences

This keeps the repository implementation-focused while preserving its philosophical foundation.

Public-facing claims should distinguish:

- Luminalia: the light
- Responsibility Pathway: the path
- Responsibility Pathway Engineering: the implementation and formalization discipline
