# ADR-0003: Repository as Responsibility Pathway

## Status

Accepted

## Context

Responsibility Pathway Engineering should not only describe responsibility pathways. The repository itself should preserve one.

Readers should be able to return from claims to definitions, from definitions to specifications, from specifications to formalization, and from formalization to assumptions.

## Decision

This repository is operated as a Responsibility Pathway.

The repository structure should preserve return paths among:

- philosophy
- definitions
- specifications
- governance
- development process
- decision records
- lessons learned
- formalization
- examples
- public claims

## Consequences

The repository is not merely storage.

Its structure is part of the project argument: responsibility can remain recoverable when the pathway is preserved.

Operational failures, when understood, should be recorded as design knowledge rather than hidden as accidental noise.
