# Academic Paper Restart Boundary

This note records a narrow restart boundary for academic paper work connected to Responsibility Pathway Engineering.

## Purpose

Academic paper work may use this repository as the current source for definitions, boundaries, examples, checker scope, formalization scope, public-source lineage, and non-goals.

The paper should help external readers inspect, compare, critique, and improve the work.

It should not turn repository notes into unsupported maturity, certification, standardization, legal, safety, compliance, fairness, or production-readiness claims.

## Restart rule

Before drafting or revising a paper section, read the current repository anchors for the claim being made.

Use repository documents as support only when the claim remains within their stated scope.

Prefer narrow paper claims that can be traced back to:

- `docs/definition.md`
- `docs/overview.md`
- `docs/repair-model.md`
- `docs/recovery-pathway-reading.md`
- `docs/example-index.md`
- `docs/checker-coverage.md`
- `docs/validator-boundary.md`
- `formal/lean/README.md`
- `docs/standardization-strategy.md`
- `docs/development-cadence-and-public-question-boundary.md`

## Paper and repository separation

The repository is the source for current RPE specification state.

A paper draft may interpret, compare, and explain the repository, but it is not itself repository certification.

A paper draft should not be used to justify new repository claims unless those claims are separately added to the repository with boundaries, examples, or review notes.

## Cleanup and duplication rule

Do not create a new paper-support document if an existing repository document already carries the needed boundary.

If a paper-support note becomes redundant, shorten, merge, archive, or delete it only after a replacement reader path is clear.

## Boundary

This note is a restart boundary for paper work only.

It is not a paper draft, abstract, submission plan, novelty proof, literature review, external review result, certification, conformance claim, standardization status, legal review, safety review, compliance review, fairness review, production-readiness claim, or AI final-responsibility transfer.
